Etapas do projeto:

Implementar o mesmo projeto feito no curso de ASP utilizando a XML nas funcionalidades a seguir:

  1. Geração do menu – Utilize a XML para representar o menu do sistema. Crie uma XSL para transformar a XML para HTML do menu. Esse código HTML pode ser embutido na página ASP utilizada para conter o menu.
  2. SQL – Não utilize código SQL espalhado pelas páginas ASP. Ao invés disso utilize XML. Por exemplo, se tiver um SQL de consulta numa página ASP, crie um método para retornar o resultado da consulta no formato XML e utilize o XML pela página ASP. Para o caso de atualização, ao invés de diretamente utilizar ADO ou SQL para fazer a atualização, crie um método para receber XML montada pela página ASP para então incluir os dados dentro da XML na base, utilizando o ADO ou a SQL.
  3. HTML – Não crie HTML diretamente nas páginas ASP. Para cada página ASP utilize uma XSL para gerar a HTML necessária para montar a página final. Por exemplo, no caso de uma consulta, ao invés de mostrar dados diretamente na página ASP, utilize a XML retornada pela consulta e uma XSL para gerar o código HTML.

Objetivos do projeto:

  1. Demonstrar a forma Modelo 2 de fazer aplicações aonde a página ASP representa o controlador, XML representa o modelo e XSL representa a visão.
  2. Utilizar XML para casos aonde a sua utilização facilita a manutenção do sistema (no caso de menu, por exemplo).
  3. Não utilizar a SQL espalhada pela aplicação toda, porque a sua utilização torna o aplicativo muito dependente a um tipo de base de dados.
