Modelo-Visão-Controlador (MVC)

 

Modelo

O modelo contém a funcionalidade principal do aplicativo. O modelo encapsula o estado do aplicativo. As vezes, como no caso de base de dados, a única funcionalidade contida é o estado. O modelo desconhece a visão e o controlador.

 

Visão

A visão prove a apresentação do modelo. Isso representa a visão do aplicativo. A visão tem acesso a dados do modelo, mas não pode atualizar o modelo. A visão não tem conhecimento do controlador. A visão precisa ser avisado quando o modelo for alterado.

 

Controlador

O controlador reage a entrada do usuário. O controlador cria e atualiza o modelo.

 

Exemplo

Considere o caso de um relógio. A função central de qualquer relógio é manter o horário. Isso é o modelo. A visão é a forma que o relógio é mostrado. Visão obtém seus dados do modelo. O controlador recebe entrada do usuário a partir do teclado ou mouse e atualiza o modelo. A atualização do modelo altera a visão.
